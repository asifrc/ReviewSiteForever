Tech Tip of the Day:

Do not delete a Heroku App when there is a significant userbase. The url is made available again to Heroku and someone else can claim it, potentially stealing your users' information. Given that heroku permits piggyback ssl, there will be no obvious signs that the site is not authentic, if the visual representation of the original site is reproduced. This can often be done easily by using the Wayback Machine if search engine crawling is not deliberately disabled on the original site.
